# DDD 战略设计：限界上下文与上下文映射

## 1. 通用语言

团队内共享的统一术语，确保业务专家和开发者的沟通无歧义。

## 2. 限界上下文

每个限界上下文是模型的边界，内部有统一的通用语言：

```
订单上下文（Order Context）
  → Order, LineItem, ShippingAddress

库存上下文（Inventory Context）
  → Product, Warehouse, StockLevel

支付上下文（Payment Context）
  → Payment, Invoice, Refund
```

## 3. 上下文映射模式

| 模式 | 关系 | 场景 |
|------|------|------|
| 合作关系 | 双向依赖 | 两个团队协同开发同一功能 |
| 共享内核 | 共享核心模型 | 通用数据字典或领域概念 |
| 客户-供应商 | 单向依赖 | API 提供方与消费方 |
| 防腐层 | 隔离外部系统 | 集成遗留系统或第三方服务 |
| 开放主机服务 | 发布 API | 对外暴露的服务接口 |
| 分离方式 | 无依赖 | 完全独立的两个上下文 |

## 4. 实战建议

- 从业务事件出发识别限界上下文
- 用上下文映射图可视化系统边界
- 防腐层是集成外部系统的关键模式
