# 技术简历撰写最佳实践

## 1. STAR 法则量化成果

| 差 | 好 |
|----|----|
| 负责后端开发 | 独立设计并实现了高并发订单系统，QPS 从 200 提升至 5000 |
| 优化数据库性能 | 通过索引优化和缓存策略，将接口响应时间从 2s 降低至 50ms |
| 参与团队管理 | 带领 5 人团队，在 3 个月内交付 2 个核心项目 |

## 2. 技术栈呈现

```
技术栈：Go / Python · PostgreSQL / Redis · K8s / Docker · gRPC / REST
```

按熟练度分组，突出核心技能。

## 3. 项目描述公式

**项目名称 + 技术栈 + 职责 + 量化成果**

示例：

> **订单中台系统**（Go + PostgreSQL + Kafka + K8s）
> 负责订单核心链路设计与开发，通过引入缓存和异步处理，将下单成功率从 95% 提升至 99.9%，日均处理订单 50 万+。

## 4. 排版原则

- 一页为佳，不超过两页
- 使用 PDF 格式，不要用 Word
- 字体统一，间距适中
- 按时间倒序排列经历

## 5. 常见错误

- ❌ 写上"精通"但经不起深问
- ❌ 罗列技术名词没有上下文
- ❌ 语法错误或格式混乱
- ❌ 包含无关个人信息（身高、体重等）
- ✅ 针对不同岗位定制简历
