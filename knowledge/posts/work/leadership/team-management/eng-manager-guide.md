# 技术管理者转型指南

## 1. 角色转变

| 工程师 | 技术管理者 |
|--------|-----------|
| 自己写代码 | 帮助团队写代码 |
| 个人产出 | 团队产出 |
| 技术深度 | 技术广度 + 管理技能 |
| 被分配任务 | 分配任务 |

## 2. 1对1沟通

结构化 1对1 模板：

```markdown
# Weekly 1:1
## 工作进展
- 本周完成的工作？
- 遇到什么阻塞？

## 成长发展
- 最近学了什么新技能？
- 需要什么资源支持？

## 反馈
- 团队有什么可以改进的？
- 对我有什么建议？
```

## 3. 团队建设

- **目标对齐**：确保每个成员理解团队目标
- **授权与信任**：给成员自主权，不过度微观管理
- **建设性反馈**：及时、具体、可执行的反馈
- **激励机制**：公开表扬，私下批评，为团队争取利益

## 4. 工程文化

- **Code Review**：建立高质量代码标准
- **技术分享**：定期内部分享，鼓励知识沉淀
- **事故复盘**：不追责，关注系统改进
- **技术债务**：预留时间偿还技术债

## 5. 管理书籍推荐
