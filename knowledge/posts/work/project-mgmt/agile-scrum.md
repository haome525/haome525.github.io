# Scrum 敏捷开发实践

Scrum 是目前最流行的敏捷开发框架，适用于复杂产品的迭代开发。

## 1. Scrum 框架

### 1.1 三个角色

- **Product Owner**：负责产品待办列表，最大化产品价值
- **Scrum Master**：确保团队遵循 Scrum 实践，移除障碍
- **开发团队**：自组织的跨职能团队，负责交付完成增量

### 1.2 五个事件

1. **Sprint**：固定时间盒（1-4 周），交付一个增量
2. **Sprint Planning**：规划本 Sprint 的工作内容
3. **Daily Scrum**：每日 15 分钟站会，同步进度
4. **Sprint Review**：展示完成的工作，收集反馈
5. **Sprint Retrospective**：回顾改进过程

### 1.3 三个工件

- **Product Backlog**：产品待办列表（优先级排序）
- **Sprint Backlog**：当前 Sprint 要完成的工作
- **Increment**：可交付的产品增量

## 2. 实践要点

### 2.1 用户故事

使用用户故事描述需求：

```
作为 [用户角色]
我想要 [功能描述]
以便 [商业价值]
```

### 2.2 估算方法

使用 Planning Poker（计划扑克）进行相对估算：
- 使用斐波那契数列（1, 2, 3, 5, 8, 13, 21）
- 团队共同讨论和估算
- 避免个人偏见

## 3. 常见误区

- ❌ Scrum 是方法论，不是银弹
- ❌ Daily Scrum 不是状态汇报
- ❌ PO 不应干涉团队如何工作
- ✅ 团队应持续改进，而不是机械执行

## 4. 推荐工具

- Jira / Azure DevOps：项目管理
- Confluence / Wiki：文档协作
- Miro / Mural：在线白板回顾
